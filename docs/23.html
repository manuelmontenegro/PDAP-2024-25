Agentes

¿Qué es un agente?

  • Un agente es un proceso que almacena un valor de Elixir (estado).

  • Existen funciones que permiten obtener el estado del agente, y modificar el estado del agente.

  • La biblioteca estándar de Elixir proporciona el módulo Agent, que implementa este tipo de procesos.

Creación de un agente

Agent.start(fn_initial_state, options \\ []) Agent.start_link(fn_initial_state, options \\ [])

  • Las funciones start y start_link reciben una función sin parámetros. El resultado de esa función determina el estado inicial del servidor.

    > Agent.start_link(fn -> 0 end)   Estado inicial: 0

  • Las opciones (options) son las mismas que en GenServer.

  • Devuelve una tupla {:ok, pid}, donde pid es el PID del agente creado.

Acceso y modificación del estado

Agent.get(agent, fun, timeout \\ 5000)

  • Devuelve el resultado de aplicar fun sobre el estado actual del agente pasado como parámetro.

Agent.update(agent, fun, timeout \\ 5000)

  • Actualiza el estado del agente, aplicándole la función fun pasada como parámetro.

    • Esta función es síncrona: espera (hasta timeout ms.) a que el servidor confirme la actualización del estado. Existe una variante asíncrona (cast).

Ejemplo

> Agent.start_link(fn -> 0 end, name: :contador)
{:ok, #PID<0.106.0>}

> Agent.get(:contador, fn x -> x end)
0

> Agent.get(:contador, &to_string/1)
"0"

> Agent.update(:contador, fn x -> x + 3 end)
:ok

> Agent.get(:contador, &(&1))
3

Otras funciones sobre agentes

  • Agent.get_and_update(agent, fun, timeout \\ 5000)

  • Versiones alternativas de las funciones anteriores, pero recibiendo tupla {mod, fun, args} en lugar de función anónima.

    • Agent.start(mod, fun, args, options)
    • Agent.start_link(mod, fun, args, options)
    • Agent.get(agente, mod, fun, args, timeout)
    • Agent.update(agente, mod, fun, args, timeout)

Precauciones sobre el uso de agentes

  • Los agentes son, esencialmente, variables globales.
    • ¡No abusar!
  • Las funciones que se pasan como parámetro a start, get, update, etc., se ejecutan en el proceso del agente.
    • Por tanto, bloquean el agente mientras se están ejecutando.
    • Se recomienda que estas funciones no sean computacionalmente complejas.

Tareas

Tareas

  • Una tarea es un proceso que realiza un determinado cómputo en segundo plano.

  • La función Task.async(fun) crea una tarea que ejecuta la función fun en un proceso separado.

    • Devuelve un struct con la información de la tarea creada.

  • Cuando deseemos obtener el resultado de la fun ejecutada en la tarea t, podemos utilizar Task.await(t).

Ejemplo

  • Suponemos que fib(n) es una función que calcula el n-ésimo número de la sucesión de Fibonacci de manera recursiva.
> t = Task.async(fn -> fib(40) end)
%Task{...}

> ...     Se pueden ir realizando otras operaciones desde la shell 

> Task.await(t)
102334155

await vs yield

Task.await(task, timeout \\ 5000)

Task.yield(task, timeout \\ 5000)

  • Ambas esperan una cantidad timeout de milisegundos a que la tarea task finalice.
    • Se permite el valor :infinity como timeout.
  • Si la tarea no ha terminado en ese lapso de tiempo:
    • await provoca salida del proceso llamante.
    • yield devuelve nil.

Aplicación: parallel_map

def parallel_map(enumerable, f) do
  enumerable
  |> Enum.map(&Task.async(fn -> f.(&1) end))
  |> Enum.map(&Task.await(&1, :infinity))
end

o, equivalentemente,

def parallel_map(enumerable, f) do
  enumerable
  |> Enum.map(&Task.async(fn -> f.(&1) end))
  |> Task.await_all(:infinity)
end

Otras funciones sobre tareas

  • Task.start_link(fun)

    Ejecuta función de modo asíncrono, pero no se espera utilizar await ni yield sobre el resultado.

  • Task.async_stream(enumerable, fun, options \\ [])

    Construye un Stream perezoso con el resultado de aplicar concurrente la función fun a cada elemento del enumerable.
