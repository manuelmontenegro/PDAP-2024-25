Funciones simples y módulos

Definición de una función

def cuadrado(x) do
  x * x
end
def suma_cuadrados(x, y) do
  cuadrado(x) + cuadrado(y)
end

  • En general, una función se define del siguiente modo:

    def nombre_funcion(arg1, ..., argn) do
  expresion
end

Definición de un módulo

  • Toda función ha de estar definida dentro de un módulo, definido mediante defmodule.
# Fichero: funciones.ex   Comentarios comienzan por # 
        
         Nombre del módulo  
defmodule FuncionesSimples do  

  def cuadrado(x) do
    x * x
  end

  def suma_cuadrados(x, y) do
    cuadrado(x) + cuadrado(y)
  end

end

Sintaxis one-liner

  • Si el cuerpo de una función es relativamente simple, puede abreviarse su definición:

Sin abreviar:

def suma(x, y) do
  x + y
end

Abreviada:

def suma(x, y), do: x + y

Con la sintaxis abreviada:

  • Coma (,) antes de do
  • Dos puntos (:) después de do
  • No se pone end

  • Esto se aplica a cualquier estructura de Elixir que haga uso de do:

    (defmodule, def, if, case, etc.)

cualquier_cosa_1 do
  cualquier_cosa_2
end

equivale a

cualquier_cosa_1, do: cualquier_cosa_2

Llamadas a funciones

  • Si la función llamada está dentro del mismo módulo, basta con indicar el nombre de la función seguido de sus argumentos:
defmodule FuncionesSimples do  

  def cuadrado(x) do
    x * x
  end

  def suma_cuadrados(x, y) do
    cuadrado(x) + cuadrado(y)      Llamadas dentro del mismo módulo 
  end

end
  • Si la función llamada está en otro módulo, debe ir precedida por el nombre de dicho módulo.
defmodule Modulo1 do  
  def cuadrado(x) do
    x * x
  end
end

defmodule Modulo2 do
  def suma_cuadrados(x, y) do
    Modulo1.cuadrado(x) + Modulo1.cuadrado(y)      Llamadas a Modulo1 
  end
end

Funciones privadas (defp)

  • Si definimos una función mediante defp, la función no será visible desde fuera del módulo:
defmodule FuncionesSimples do  

  defp cuadrado(x) do              Función privada 
    x * x
  end

  def suma_cuadrados(x, y) do      Función pública 
    cuadrado(x) + cuadrado(y)     
  end

end

La máquina virtual de Erlang (BEAM)

Compilación de ficheros en Elixir

  • Los ficheros de Elixir se compilan a código bytecode que se ejecuta en la máquina virtual de Erlang (BEAM).

  • Para cada módulo, el compilador genera un fichero .beam con el bytecode del mismo.

    • Este fichero puede ser ejecutado por la máquina virtual de Erlang.

  • Dentro de un mismo fichero Elixir (.ex) pueden definirse varios módulos.

Llamada a funciones de Elixir

  • Cuando llamamos a una función f en Elixir definida dentro de un módulo M:

    M.f(...)

    Se espera que el bytecode de la función f se encuentre en el fichero Elixir.M.beam.

Interoperabilidad con Erlang

  • Es posible llamar a funciones de Erlang desde Elixir.

  • Basta con indicar el nombre del módulo Erlang precedido por dos puntos (:).

    :mi_mod.f()

    En este caso, se espera que el bytecode de f esté definido en el fichero mi_mod.beam.

Ejemplo

-module(mod_cuadrado).

-export([cuadrado/1]).

cuadrado(X) -> X * X.
defmodule ModuloSumaCuadrados do
  def suma_cuadrados(x, y) do
    :mod_cuadrado.cuadrado(x) * :mod_cuadrado.cuadrado(y)
  end
end

Ejecución de funciones en Elixir

El shell interactivo iex

  • El intérprete iex permite compilar ficheros de Elixir y evaluar expresiones.
Interactive Elixir (1.17.1) - press Ctrl+C to exit (type h() for help)

iex(1)> 2 + 2
4
iex(2)> String.length("Adios")
5
iex(3)> to_string(23)
"23"

Compilación de módulos

  • Es posible definir módulos directamente en la shell,

    iex(5)> defmodule MiMod do
...(5)>   def cuadrado(x), do: x * x
...(5)> end

iex(6)> MiMod.cuadrado(5)
25

…pero lo habitual es compilarlos de un fichero .ex:

# Fichero: funciones.ex

defmodule FuncionesSimples do  
  defp cuadrado(x) do
    x * x
  end

  def suma_cuadrados(x, y) do
    cuadrado(x) + cuadrado(y)
  end
end

En iex:

iex(7)> c "funciones.ex"
[FuncionesSimples]     Nombres de los módulos cargados 
iex(8)> FuncionesSimples.suma_cuadrados(3, 4)
25

F.A.Q.

  • ¿Cómo se sale de iex?

    Pulsando Ctrl+C dos veces.

  • ¿Tenemos que poner siempre el nombre del módulo cuando llamamos a una función desde la shell?

    No es necesario si se utiliza import previamente:

    > c "funciones.ex"
[FuncionesSimples]
> import FuncionesSimples
> suma_cuadrados(3, 4)     ➀
25

Scripts en Elixir

  • Elixir también es un lenguaje de scripting.
  • Los ficheros de script tienen la extensión .exs
# Fichero suma_cuadrados.exs

defmodule FuncionesSimples do  
  defp cuadrado(x) do
    x * x
  end

  def suma_cuadrados(x, y) do
    cuadrado(x) + cuadrado(y)
  end
end

IO.puts("El resultado es:")      IO.puts imprime por la salida estándar 
IO.puts(FuncionesSimples.suma_cuadrados(3, 4))

  • Para ejecutar los script, se utiliza el intérprete elixir desde la línea de comandos:
$ elixir suma_cuadrados.exs
El resultado es:
25

  • También se puede incorporar la siguiente línea shebang al fichero .exs para poder ejecutar el fichero directamente en GNU/Linux:

    #!/usr/bin/env elixir

# Fichero suma_cuadrados.exs

defmodule FuncionesSimples do  
...

  • Desde la línea de comandos:

    $ chmod u+x suma_cuadrados.exs
$ ./suma_cuadrados.exs
El resultado es:
25

Definiciones y control de flujo

Definiciones auxiliares

  • Es posible definir definiciones auxiliares en el cuerpo de las funciones:
def suma_cuadrados(x, y) do
  cuad_x = x * x
  cuad_y = y * y
  cuad_x + cuad_y
end
  • Al operador = se le llama operador de ajuste (matching).

  • En Elixir, las variables son inmutables. Una vez ligadas a un valor, no es posible cambiarlo.

  • Aunque Elixir permite cosas como esta,

    iex(1)> x = 3
iex(2)> x = 5 * x
iex(3)> x
15

    cuando se hace x = 5 * x no se está sobreescribiendo el valor de la variable x, sino que se crea una nueva variable que tiene el mismo nombre que la anterior.

Mecanismos de control de flujo

  • Guardas
  • Expresiones if
  • Expresiones cond
  • Ajuste de patrones simple
  • Ajuste de patrones mediante case (Semana 2)

Guardas

  • Es posible definir una función mediante varias cláusulas, cada una con una condición:
def signo(x) when x < 0 do
  -1
end

def signo(x) when x == 0 do
  0  
end

def signo(x) when x > 0 do
  1
end
  • No son tres definiciones de signo. Es una definición con tres casos.
  • Se evaluará la primera cláusula cuya guarda sea true:
signo(-3)    # se evalúa a -1 
signo(4)     # se evalúa a 1 
signo(0)     # se evalúa a 0

  • Si varias guardas se cumplen para un determinado valor, se evalúa solamente la primera de ellas.

    def signo_string(x) when x >= 0, do: "Positivo"
def signo_string(x) when x <= 0, do: "Negativo"
    signo_string(0)     # se evalúa a "Positivo"

  • Si ninguna guarda toma el valor cierto, se produce un error:

    def signo_string(x) when x > 0, do: "Positivo"
def signo_string(x) when x < 0, do: "Negativo"
    signo_string(0)  # ERROR: no function clause matching in signo_string

  • Si la última cláusula no tiene guarda, se ejecutará solamente si ninguna de las cláusulas anteriores se ha ejecutado:

    def signo_string(x) when x > 0, do: "Positivo"
def signo_string(x), do: "No positivo"
    signo_string(3)   # "Positivo"
signo_string(-1)  # "No positivo"
signo_string(0)   # "No positivo"
  • A esta última cláusula se le denomina normalmente catch-all.

Limitaciones de las guardas

  • Desde una guarda solo se puede llamar a un conjunto limitado de funciones predefinidas en el lenguaje.

  • Esto se hace para evitar que la evaluación de una guarda tenga efectos laterales arbitrarios.

  • Por tanto, no se permite lo siguiente:

          Incorrecto: función no predefinida 
def func(x) when suma_cuadrados(x, x) > 10, do: ...

¿Qué funciones se permiten en las guardas?

  • Operadores aritméticos: +, -, *, div, rem, etc.

  • Comparaciones: ==, !=, <=, etc.

  • Operadores lógicos: or, and, not

  • Funciones de comprobación de tipos: is_integer, is_list, etc.

  • Algunas otras funciones: elem, in, length, etc.

Ajuste de patrones simple

  • Cuando una guarda tiene la forma x == valor (donde x es un parámetro) se puede colocar directamente el valor en el parámetro.

Podemos escribir

def signo(x) when x < 0, do: -1
def signo(0), do: 0
def signo(x) when x > 0, do: 1

en lugar de

def signo(x) when x < 0, do: -1
def signo(x) when x == 0, do: 0
def signo(x) when x > 0, do: 1
  • El ajuste de patrones es una herramienta muy potente en programación funcional. Más adelante veremos otras aplicaciones.

Expresiones if

if condicion do
  expr1
else
  expr2
end

Ejemplo:

def abs(x) do
  if x >= 0 do
    x
  else
    -x
  end
end

  • En Elixir, if es una expresión, y no una sentencia.

  • Podemos utilizar expresiones if dentro de otras expresiones.

    x = if y <= 0 do
      0
    else
      y
    end

  • También existe una forma one-liner para el if:

    x =  if y <= 0, do: 0, else: y
  • Al contrario que en las guardas, en un if es posible utilizar cualquier expresion, aunque haga llamadas a funciones definidas por el usuario:
def suma_cuadrados(x), do: ...

def en_circulo_unidad(x) do
  if suma_cuadrados(x) >= 1 do
    1
  else
    -1
  end
end

Expresiones cond

  • Son una alternativa a las cadenas if-else if:

    cond do
  condicion1 -> expr1
  condicion2 -> expr2
  ...
  condicionn -> exprn
end

  • Ejemplo:

    def signo(x) do
  cond do
    x < 0  -> -1
    x == 0 -> 0
    x > 0  -> 1
  end
end

  • Al igual que los if, los cond son expresiones. Pueden utilizarse dentro de otras expresiones.

  • Si ninguna de las condiciones del cond se cumple, se produce un error en tiempo de ejecución.

  • Se puede añadir una alternativa catch-all con la condición true, en caso de ser necesario:

    def tipo_usuario(fecha_nacimiento) do
  edad = calcular_edad(fecha_nacimiento)
  cond do
    edad >= 67 -> "Jubilado"
    edad >= 18 and edad < 67 -> "Adulto"
    true -> "Infantil"   Cláusula catch-all 
  end
end

Más sobre módulos y funciones

Anidamiento de módulos

  • Los módulos de Elixir pueden anidarse:

    defmodule ModA do 
  defmodule ModB do    Módulo anidado 
    def incr(x), do: x + 1
  end
end

  • El nombre del módulo interno viene precedido por el del módulo externo, separados por un punto (.).

    iex(1)> ModA.ModB.incr(3)
4

  • El anidamiento puede ser «virtual», en el sentido en que el módulo interno puede declararse de manera independiente:
defmodule ModA do
  ...
end

defmodule ModA.ModB do
  def incr(x), do: x + 1
end

Aridad de funciones

  • En Elixir (y Erlang), es posible definir varias funciones con el mismo nombre, siempre que difieran en su aridad.

    aridad = número de parámetros 
defmodule Aridad do
  # Máximo de dos parámetros
  def maximo(x, y) when x <= y, do: y
  def maximo(x, y), do: x

  # Máximo de tres parámetros
  def maximo(x, y, z)  do
    maximo(x, maximo(y, z))
  end
end
iex(1)> Aridad.maximo(-1, 3)
3
iex(2)> Aridad.maximo(5, -1, 3)
5

  • En la documentación, para evitar ambigüedades, se suele indicar la aridad de las funciones mediante la notación func/n, donde func es el nombre de la función y n es su aridad.

  • Ejemplo:

    «Si queremos calcular el máximo de tres números, podemos utilizar la función maximo/3. También existe una variante maximo/2 que recibe solo dos números»

  • Ejemplo en la documentación de Elixir:

    Fuente: https://hexdocs.pm/elixir/Enum.html

Parámetros por defecto

  • El tener una función con distintas aridades nos permite implementar un mecanismo de parámetros por defecto en funciones.

  • Por ejemplo, supongamos una función precio_final(precio, descuento) que aplique el porcentaje descuento al precio pasado como parámetro.

  • Si no se indica descuento, se supondrá que es del 10 %.

def precio_final(precio, descuento) do
  precio - (precio * descuento / 100)
end

def precio_final(precio) do
 Llamamos a precio_final/2 
  precio_final(precio, 10)
end
> precio_final(50, 25)
37.5
> precio_final(50)
45.0

  • Sin embargo, Elixir tiene una sintaxis abreviada para definir valores por defecto en parámetros:
                  Valor por defecto para el segundo parámetro 
def precio_final(precio, descuento \\ 10) do
  precio - (precio * descuento / 100)
end
  • Esto genera dos definiciones de funciones: precio_final/2 y precio_final/1.

  • Si una función esta definida mediante múltiples cláusulas, los parámetros por defecto se indican una sola vez, antes de las cláusulas:
def precio_final(precio, descuento \\ 10)

def precio_final(precio, descuento) when descuento >= 0 do
  precio - (precio * descuento / 100)
end
def precio_final(precio, descuento) when descuento < 0 do
  precio
end

Convenio de nombres

  • Los nombres de módulos comienzan por letra mayúscula y utilizan el convenio CamelCase:

    Ejemplos: OptionParser, GestorFinanzas, Base

  • Los nombres de variables y funciones comienzan por letra minúscula y utilizan el convenio snake_case:

    Ejemplos: spawn_monitor, to_string, sueldo_base

  • Las funciones que devuelven un valor booleano suelen llevar el signo ? al final de su nombre:

    Ejemplos: String.contains?, List.starts_with?, etc.

def positivo?(x) do
  x > 0
end
> positivo?(4)
true
> positivo?(-3)
false
  • Excepción: funciones de comprobación de tipos (is_integer, is_list, etc.)

  • Si el nombre de una función finaliza en !, significa que esa función puede lanzar una excepción en casos de fallo.

  • Normalmente se utiliza cuando hay dos versiones de una misma función:

    • File.read("ejemplo.txt")

      Devuelve un valor especial (:error) si el fichero ejemplo.txt no se puede leer.

    • File.read!(“ejemplo.txt”)

      Lanza una excepción si el fichero ejemplo.txt no se puede leer.
