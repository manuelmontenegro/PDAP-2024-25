Funciones recursivas

¿No hay bucles?

  • En los lenguajes funcionales no existe el concepto de bucle que podemos encontrar en el paradigma imperativo (while, for, etc.)

¿Por qué?

  • Una de las características programación funcional pura es la inmutabilidad de los valores.

    • Las variables, una vez asignadas a un valor, no pueden ser modificadas.

    • No solo se aplica a las variables: todos los tipos de datos (listas, tuplas, maps, etc.) son inmutables.

  • Sin embargo, los bucles necesitan modificar los valores de las variables:

    int x = 1, i = 1;
while (i <= 10) {
  x = x * i;
  i = i + 1;
}

Recursión como alternativa

  • La alternativa a los bucles en programación funcional son las funciones recursivas.

  • Una función recursiva es una función que se llama a sí misma.

def factorial(0), do: 1    Caso base 

def factorial(n) do        Caso recursivo 
  n * factorial(n - 1)  Llamada recursiva 
end

Evaluamos factorial(3):

factorial(3)

➤ 3 * factorial(2)

➤ 3 * (2 * factorial(1))

➤ 3 * (2 * (1 * factorial(0)))    Caso base 

➤ 3 * (2 * (1 * 1))

➤ 3 * (2 * 1)

➤ 3 * 2

➤ 6

Evaluamos factorial(-3):

factorial(-3)

➤ -3 * factorial(-4)

➤ -3 * (-4 * factorial(-5))

➤ -3 * (-4 * (-5 * factorial(-6)))

➤ -3 * (-4 * (-5 * (-6 * factorial(-7))))

➤ ...

¡La recursión no termina, porque nunca llega al caso base!

  • Podemos evitar la no terminación añadiendo una guarda al caso recursivo:
def factorial(0), do: 1

def factorial(n) when n > 0 do
  n * factorial(n - 1)
end
> factorial(5)
120
> factorial(-3)
** (FunctionClauseError) no function clause matching in factorial/1

Otros ejemplos

  • pow(x, n): calcula el valor de xn.

    def pow(x, 0), do: 1
def pow(x, n) when n > 0 do
  x * pow(x, n - 1)
end

  • contar(ini, fin): imprime varias líneas con el mensaje Numero X, donde X recorre los valores de ini a fin.

    def contar(ini, fin) when ini > fin, do: :ok
def contar(ini, fin) when ini <= fin do
  IO.puts("Número #{ini}")
  contar(ini + 1, fin)
end

El precio de la recursión

  • Durante la ejecución de un programa se almacena una pila de llamadas a función (call stack).

  • Por ejemplo, dado el siguiente programa:

    def m(), do: h(3)
def h(x), do: g(x + 1) * 2
def g(x), do: f(x) - 3
def f(x), do: x + 1

    supongamos la siguiente llamada:

    > m()
4

  • Al ejecutar la expresión m(), en el momento de evaluarse la llamada a f, tenemos la siguiente pila de llamadas.

    • En la cima de la pila está la función que se está ejecutando actualmente (f).
    • En la subcima está la función que llamó a f, y así sucesivamente.
 
f(4) Función ejecutándose actualmente
g(4) Función que llama a f
h(3) Función que llama a g
m() Llamada inicial
 
f(4)
g(4)
h(3)
m()
  • A cada elemento de la pila se le llama frame.
  • Cada frame contiene los valores de los parámetros y definiciones locales correspondientes a esa llamada.
  • Cada vez que se llama a una función se inserta un nuevo frame en la pila.
  • Cuando una función finaliza, se elimina su frame de la pila
  • Por ejemplo, la evaluación de factorial(4) genera la siguiente pila de llamadas:
 
factorial(0)
factorial(1)
factorial(2)
factorial(3)
factorial(4)

  • El tamaño de la pila de llamadas tiene un límite.

  • Si la cadena de llamadas es muy grande, se puede superar ese límite de tamaño, provocando un desbordamiento de pila (Stack Overflow).

  • ¿Cómo puede evitarse el desbordamiento de pila? Mediante la recursión de cola.

Recursión de cola

Llamadas recursivas finales

  • Decimos que una llamada recursiva es final si no se realiza ninguna operación tras esa llamada.

(excepto devolver el valor devuelto por esa llamada)

def f(x) do
  cond do
    x == 0 -> :ok
    x == 1 -> y = f(10)        Llamada no final 
              y * 2
    x > 1  -> 1 + f(x - 1)     Llamada no final 
    x < 0  -> z = x + 1
              f(z)             Llamada final 
  end
end

Funciones recursivas de cola

  • Decimos que una función tiene recursión de cola si todas sus llamadas recursivas son finales.

  • La función f de la diapositiva anterior no tiene recursión de cola, porque tiene algunas llamadas recursivas que no son finales.

def f(x) do
  cond do
    x == 0 -> :ok
    x == 1 -> y = f(10)
              y * 2
    x > 1  -> 1 + f(x - 1)
    x < 0  -> z = x + 1
              f(z)
  end
end

Ejemplos

  • La función factorial definida anteriormente no es recursiva de cola:

    def factorial(0), do: 1

def factorial(n) when n > 0 do
  n * factorial(n - 1)     Llamada no final 
end

  • La función pow tampoco contiene recursión de cola.

      def pow(x, 0), do: 1
  def pow(x, n) when n > 0 do
    x * pow(x, n - 1)      Llamada no final 
  end

  • La función contar sí tiene recursión de cola:

      def contar(ini, fin) when ini > fin, do: :ok
  def contar(ini, fin) when ini <= fin do
    IO.puts("Número #{ini}")
    contar(ini + 1, fin)     Llamada final 
  end

  • Pero la siguiente variante, que se comporta exactamente igual, no tiene recursión de cola:

      def contar(ini, fin) when ini > fin, do: :ok
  def contar(ini, fin) when ini <= fin do
    contar(ini, fin - 1)     Llamada no final 
    IO.puts("Número #{fin}")
  end

¿Para qué sirve la recursión de cola?

  • Hemos visto que, cada vez que se hace una llamada a una función, se inserta un nuevo frame en la pila de llamadas:
 
 
 
 
 
contar(1, 10)

➤

 
 
 
 
contar(2, 10)
contar(1, 10)

➤

 
 
 
contar(3, 10)
contar(2, 10)
contar(1, 10)
  • Si la función es recursiva de cola, cada frame puede sobreescribir al anterior.
  • El tamaño de la pila se mantiene constante, por lo que la función puede hacer un número ilimitado de llamadas recursivas sin que haya desbordamiento de pila.
 
 
 
 
 
contar(1, 10)

➤

 
 
 
 
 
contar(2, 10)

➤

 
 
 
 
 
contar(3, 10)

Parámetros acumuladores

Parámetros acumuladores

  • Los parámetros acumuladores permiten transformar una función en otra recursiva de cola equivalente.

  • Un parámetro acumulador es un parámetro en el que se va construyendo el resultado final de la función.

  • Cuando la ejecución llega al caso base, se devuelve el valor del parámetro acumulador.

Ejemplo: factorial

  • Sin parámetro acumulador:

    def factorial(0), do: 1
def factorial(n) when n > 0 do
  n * factorial(n - 1)
end

  • Con parámetro acumulador:

    def factorial_acum(0, ac), do: ac     ac es el parámetro acumulador 
def factorial_acum(n, ac) when n > 0 do
  factorial_acum(n - 1, ac * n)
end

    El valor del acumulador en la llamada inicial debe ser 1 (lo que en la versión anterior era el caso base).

    > factorial_acum(5, 1)
120

¿Cómo funciona?

   factorial_acum(5, 1)

➤  factorial_acum(4, 5)      5 = 1 * 5 

➤  factorial_acum(3, 20)     20 = 5 * 4 

➤  factorial_acum(2, 60)     60 = 20 * 3 

➤  factorial_acum(1, 120)    120 = 60 * 2 

➤  factorial_acum(0, 120)    Caso base: devolver acumulador 

➤  120

Llamada inicial

  • Como el valor del parámetro acumulador en la llamada inicial siempre es el mismo, podemos encapsular la llamada inicial factorial_acum en otra función:
def factorial(n), do: factorial_acum(n, 1)   Llamada inicial 

defp factorial_acum(0, ac), do: ac     `factorial_acum` ahora es privada 
defp factorial_acum(n, ac) when n > 0 do
  factorial_acum(n - 1, ac * n)
end
> factorial(5)
120

Otro ejemplo: duplicate

  • Escribir una función que reciba una cadena str y un número n y devuelva el resultado de concatenar str consigo misma n veces.

  • Sin recursión de cola:

    def duplicate(str, 0), do: ""
def duplicate(str, n) do
  str <> duplicate(str, n - 1)
end
    > duplicate("-", 5)
"-----"

  • Con parámetros acumuladores y recursión de cola:

    def duplicate(str, n), do: duplicate(str, n, "")

defp duplicate(str, 0, ac), do: ac
defp duplicate(str, n, ac) do
  duplicate(str, n - 1, ac <> str)
end

  • La función duplicate/2 es pública, y hace la llamada inicial a duplicate/3, que es la que utiliza el parámetro acumulador.

   duplicate("-", 3)

➤  duplicate("-", 3, "")

➤  duplicate("-", 2, "-")

➤  duplicate("-", 1, "--")

➤  duplicate("-", 0, "---")

➤  "---"

Bucles y recursión

Bucles y recursión

  • Cualquier bucle de un lenguaje imperativo se puede transformar en una función recursiva de cola.

  • Por ejemplo:

    int x = 1, i = 1, n = 10;
while (i <= n) {
  x = x * i;
  i = i + 1;
}
return x;

    definimos una función con tantos parámetros acumuladores como variables intervengan en el cuerpo y en la condición del bucle.

    def bucle(x, i, n) do ...
int x = 1, i = 1, n = 10;
while (i <= n) {
  x = x * i;
  i = i + 1;
}
return x;
  • El caso base corresponde a la condición de salida del bucle:
defp bucle(x, i, n) when i > n, do: x   Resultado de la función
  • El caso recursivo corresponde a una iteración del bucle:
defp bucle(x, i, n) do
  bucle(x * i, i + 1, n)     Estado de las variables tras una iteración 
end
int x = 1, i = 1, n = 10;
while (i <= n) {
  x = x * i;
  i = i + 1;
}
return x;
  • La llamada inicial se corresponde con los valores iniciales de las variables antes de la primera iteración:
def bucle(), do: bucle(1, 1, 10)

defp bucle(x, i, n) when i > n, do: x
defp bucle(x, i, n) do
  bucle(x * i, i + 1, n)
end
