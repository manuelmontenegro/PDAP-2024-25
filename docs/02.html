Elixir es dinámicamente tipado

¿Dónde están los tipos?

Lenguajes estáticamente tipados (C++, Java, Haskell)

  • El compilador verifica los tipos de las variables en tiempo de compilación, y detecta expresiones mal formadas (por ejemplo, true + "hola").

  • El compilador puede requerir anotaciones de tipo (p.ej, C++, Java) o inferir los tipos por sí mismo:

    // Ejemplo en Java
String str = "Hola, mundo";
var x = 3;  Tipo inferido 
    -- Ejemplo en Haskell
incr :: Int -> Int   Tipo inferido 
incr x = x + 1

  • Previene errores en tiempo de ejecución, pero a costa de prohibir programas correctos.

Lenguajes dinámicamente tipados (Python, JS, Elixir)

  • Los tipos de las variables se comprueban en tiempo de ejecución.

    # Ejemplo en Python

def f(x):
  return x + 1

f("Hola")
    Traceback (most recent call last):
  File "...", line ...
    f("Hola")
  File "...
    return x + 1
           ~~^~~
TypeError: can only concatenate str (not "int")
to str

  • Mayor flexibilidad, pero errores potenciales en tiempo de ejecución.

Sistema de tipos en Elixir

  • Elixir es un lenguaje dinámicamente tipado,

  • … pero ofrece dos funcionalidades que nos ayudan a capturar errores de tipo en tiempo de compilación:

    • Especificación y comprobación de tipos mediante Dialyzer

    • Sistema de tipos gradual (en desarrollo, a partir de v1.17)

      https://hexdocs.pm/elixir/1.17.2/gradual-set-theoretic-types.html

Especificaciones de tipos

@spec incr(integer) :: integer   Especificación de tipo 
def incr(x) do
  x + 1
end

  • Existe una herramienta, Dialyzer, que utiliza las anotaciones @spec para detectar inconsistencias de tipos en los programas.

  • Incluso aunque no se utilice Dialyzer, las especificaciones de tipo son útiles a efectos de documentación.

Funciones de comprobación de tipos

Funciones de comprobación de tipos

  • Las funciones predefinidas is_[...] permiten determinar si el argumento que reciben es de un determinado tipo.

    • is_integer/1
    • is_float/1
    • is_boolean/1
    • is_atom/1
    • is_binary/1
    • is_list/1
    • is_map/1
    • etc.

    Lista completa: https://hexdocs.pm/elixir/1.17.2/Kernel.html#guards

  • Pueden utilizarse en guardas.

def incr(x) when is_number(x) do  # number = integer o float 
  x + 1
end

def incr(str) when is_binary(str) do  # binary = string 
  String.to_integer(str) + 1
end
> incr(34)
35
> incr("21")
22

Enteros

Literales enteros

  • Pueden escribirse en base decimal, hexadecimal, octal o binaria.

    235     0x3a     0o23     0b10010

  • Puede utilizarse el símbolo _ como separador. Este símbolo se ignora.

    1_000_000 equivale a 1000000

  • No hay límite en el tamaño de los enteros.

> Integer.pow(2, 200)
1606938044258990275541962092341162602522202993782792835301376

Operaciones predefinidas sobre enteros

  • Suma (+), resta (-), multiplicación (*), potencia (**)

  • División (/), división entera (div), resto (rem)

    > 4 / 2
2.0
> div(5, 2)
2

  • Máximo (max), mínimo (min), valor absoluto (abs)

El módulo Integer

> Integer.is_even(20)
true
> Integer.digits(3423)
[5, 3, 1, 3, 4]
> Integer.undigits([1, 4, 2])
142
> Integer.gcd(8, 12)
4

Ver: https://hexdocs.pm/elixir/1.17.2/Integer.html

Valores en coma flotante (floats)

Literales y funciones en coma flotante

  • Literales

    1.23     0.314e1     1.0e-4

  • Funciones

    • Aritméticas básicas: +, -, *, /, **

    • Conversión a enteros: trunc/1, ceil/1, floor/1, round/1.

    • Conversión a fracciones: Float.ratio/1

      > Float.ratio(1.5)

Átomos

¿Qué son los átomos?

  • En Elixir, los átomos son constantes simbólicas.

    • Básicamente, son constantes con un nombre asignado.

  • Son parecidos a los enum de C++ y Java:

    enum DiaSemana { LUNES, MARTES, MIERCOLES, JUEVES, ... };

    pero en Elixir no hay que declarar las constantes de antemano; se utilizan directamente.

Átomos en Elixir

  • En Elixir, los átomos se representan mediante un identificador precedido por dos puntos (:).

    • Ejemplos: :lunes, :jueves, :ok, :error_grave

    • Excepciones: true, false y nil

  • Si se utilizan caracteres especiales, o dígitos al comienzo, el nombre del átomo puede delimitarse entre comillas:

    • Ejemplos: :"lunes festivo", :"2veces"

¿Para qué se utilizan?

  • Para representar valores constantes y etiquetas (tags).

  • Los átomos se almacenan internamente como números enteros, por lo que comparar dos átomos equivale a la comparación de los enteros correspondientes.

    > :hola == :hola
true

  • Muchas funciones de Elixir (por ejemplo, las de E/S) devuelven :ok en caso de éxito.

    > IO.puts("hola")
"hola"
:ok

¿Los átomos son strings?

  • No es lo mismo un átomo que una cadena.

  • Las cadenas se delimitan entre comillas dobles (p.ej. "hola") y se utilizan para almacenar y manipular texto; no como constantes.

Tabla de átomos

  • La máquina BEAM almacena una tabla de átomos, que asocia cada átomo con el número que se utiliza internamente para representarlo.

  • Esta tabla tiene un tamaño limitado (≈ 1M entradas).

    • Es suficiente para la gran mayoría de aplicaciones, pero no conviene crear átomos dinámicamente de manera indiscriminada.

Valores booleanos

Booleanos en Elixir

  • Elixir no tiene un tipo booleano como tal.

  • Los booleanos son átomos.

  • Existen dos constantes booleanas: :true y :false que, por comodidad, pueden escribirse sin el símbolo :.

    > 1 == 0
false
> 2 < 4
true
> :true == true     `:true` y `true` son sinónimos 
true
> :false == false   `:false` y `false` son sinónimos 
true
> is_atom(false)
true

Booleanos y expresiones de control (if, cond, etc.)

  • Dentro de las expresiones if y cond, y dentro de las guardas se pueden utilizar valores no booleanos.

  • Se considera falso el valor false o el átomo nil.

  • Se considera cierto cualquier otro valor.

    > x = 2
> if x, do: "Si", else: "No"
"Si"

Operadores booleanos

  • and, or, not     Más habituales

    • Sus operandos han de ser booleanos (true o false).
    • Pueden utilizarse en guardas.

  • &&, ||, !     Menos habituales

    • Permiten otros valores, además de los booleanos:
      • false o nil se interpretan como falso, y cualquier otro valor se interpreta como cierto.
    • No se pueden utilizar en guardas.

El valor nulo (nil)

El valor nulo (nil)

  • Tambien es un átomo, pero se puede escribir sin los dos puntos (:).

    > nil == :nil
true

  • En algunas funciones, se utiliza para representar la ausencia de resultados.

    > Enum.at([1, 3, 5], 0)
1
> Enum.at([1, 3, 5], 3)
nil

Cadenas (strings)

Dos tipos de cadenas

  1. Strings propiamente dichos:

    • En realidad son binaries (cadenas de bytes) codificados mediante UTF-8.
    • Se representan mediante comillas dobles (p.ej, "hola").
    • Es la opción más recomendable en Elixir

  2. Charlists, heredados de Erlang:

    • Son listas de caracteres.
    • Se representan mediante comillas simples (p.ej, 'hola').
    • Solo se recomiendan para interoperabilidad con las funciones de Erlang.

Interpolación de cadenas

  • Mediante la sintaxis #{...} Elixir permite evaluar expresiones dentro de cadenas.

    > nombre = "Manuel"
> "Hola, #{nombre}"
"Hola, Manuel"
> edad = 99
> "Vas a cumplir #{edad + 1} años"
"Vas a cumplir 100 años"

  • Secuencias de escape habituales: \n, \t, \\, etc.

  • Concatenación de cadenas (y binaries en general): <>

    > "Hola, " <> "mundo"
"Hola, mundo"

Conversión entre cadenas y valores

  • to_string/1: convierte el valor dado en cadena:

    > to_string(12)
"12"
> to_string(:pepe)
"pepe"

  • String.to_integer/1, String.to_float/1: convierte la cadena a un valor entero o float:

    > String.to_integer("23")
23

  • También: Integer.parse/1, Float.parse/1

    • Más información: [+]

Code points y grafemas

  • Code points: partes indivisibles que conforman una cadena.
  • Grafemas: caracteres que se perciben como una única entidad, pero están formados por varios code points.

    • El grafema ñ está formado por n y ̃.

    • El grafema 👩‍🚒 está formado por 👩‍ y 🚒.

Funciones codepoints/1 y graphemes/1

  • Dada una cadena, devuelven una lista con sus grafemas o code points, respectivamente.
> String.graphemes("👩‍🚒")
["👩‍🚒"]
> String.codepoints("👩‍🚒")
["👩‍", "🚒"]

Sigils

  • Son expresiones que construyen un valor a partir de una cadena.

  • Sintaxis: ~🔵{...}

    donde 🔵 es un carácter que determina el tipo de valor a construir.

    • Si 🔵 es un carácter en minúscula, se tiene en cuenta la interpolación y caracteres de escape.

    • Si 🔵 es un caraćter en mayúscula, no se tiene en cuenta la interpolación ni los caracteres de escape.

Posibles valores de 🔵

  • ~s{...}, ~S{...}: construyen cadenas (equivale a "...").
  • ~c{...}, ~C{...}: construyen charlists (equivale a '...').
  • ~r{...}: construye expresiones regulares.
  • ~w{...}, ~W{...}: construyen listas de palabras.
  • ~D{...}: construye objetos Date.

Posibles delimitadores

  • ~🔵{...}
  • ~🔵|...|
  • ~🔵"..."
  • ~🔵'...'
  • ~🔵/.../
  • ~🔵(...)
  • ~🔵[...]
  • ~🔵<...>

Ejemplos

> IO.puts(~s{Esto es una cadena})
Esto es una cadena

> IO.puts(~s{Aquí puedo utilizar "comillas"})
Aquí puedo utilizar "comillas"

> IO.puts(~S{Las secuencias de escape \n se ignoran})
Las secuencias de escape \n se ignoran

> ~w<Cinco lobitos tiene la loba>
["Cinco", "lobitos", "tiene", "la", "loba"]

> d = ~D{2000-10-01}
> Date.add(d, 15)
~D[2000-10-16]

Cadenas multilínea (heredocs)

  • Están delimitadas por """.

  • Se utilizan normalmente para documentar funciones.

  • La cadena construida no contiene ni el salto de línea y ni la indentación inicial.

> x = """
      Esto es una cadena larga
      que ocupa varias líneas
      """

> x
"Esto es una cadena larga\nque ocupa varias líneas\n"

  • Podemos documentar funciones añadiendo el atributo @doc antes de su definición.

  • Existen herramientas (ExDoc) que generan documentación en HTML a partir de las anotaciones @doc, similar a la de la biblioteca estándar.

@doc """
Calcula el cuadrado del valor pasado como parámetro.

## Ejemplo

  iex> cuadrado(5)
  25
"""
defp cuadrado(x) do
  x * x
end

Operadores de comparación

Igualdad (==) y desigualdad (!=)

  • Permiten comparar dos valores, aunque sean de distinto tipo:

  • La igualdad entre dos elementos de distinto tipo es false, salvo si se hace entre enteros y float:

    > 1 == 1.0
true

  • Existen operadores de igualdad y desigualdad estrictos (===, !==) que no identifican los enteros con los float.

    > 1 === 1.0
false

Comparación (<, <=, >, >=)

  • Los tipos numéricos se comparan por su valor:

    1 < 2                # true
4.0 <= 3             # false

  • Los átomos se comparan por sus nombres, en orden lexicográfico:

    :alfredo < :berta    # true

  • Las cadenas se comparan byte a byte, en orden lexicográfico:

    "alfredo" < "berta"  # true
"álvaro" < "berta"   # false

  • Si se comparan elementos de distinto tipo, se utiliza el siguiente orden:

number < atom < reference < function < port < pid < tuple < map < list < bitstring

2 < :inf      # true
:inf < "inf"  # true
