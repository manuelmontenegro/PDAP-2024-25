¿Qué es el orden superior?

Orden superior

  • Decimos que un lenguaje tiene orden superior si trata a las funciones como ciudadanos de primera clase.

  • En otras palabras, permite tratar a las funciones como cualquier otro tipo de datos. En particular, se permite:

    • Pasar funciones como parámetro.
    • Devolver funciones como resultado.
    • Almacenar funciones en listas, tuplas, etc.

Ejemplo

  • Suponemos el siguiente módulo:

    defmodule FuncionesBasicas do
  def suma(x), do: x + 1

  def suma(x, y), do: x + y

  def cuadrado(x), do: x * x
end

  • Es posible asignar funciones a variables del siguiente modo:

    > f = &FuncionesBasicas.suma/1   Asignamos suma/1 a f 
> g = &FuncionesBasicas.suma/2   Asignamos suma/2 a g 
> f
&FuncionesBasicas.suma/1
> g
&FuncionesBasicas.suma/2

Referencias a funciones

  • Para hacer referencia a una función (con el fin de almacenarla en algún sitio o pasarla como parámetro) se utiliza la siguiente sintaxis:

    &Modulo.funcion/aridad

    Ejemplos: &IO.puts/1, &FuncionesBasicas.cuadrado/1.

  • Si se ha hecho previamente un import sobre el módulo, puede omitirse este último.

    > import FuncionesSimples
> f = &suma/1
> f
&FuncionesSimples.suma/1

Almacenar funciones en listas

  • Del mismo modo, es posible almacenar funciones dentro de tuplas y listas.

    > t = {&IO.puts/1, &length/1}

> elem(t, 1)
&:erlang.length/1

> xs = [&suma/1, &cuadrado/1, &to_string/1]

Llamar funciones almacenadas en variables

> f = &FuncionesBasicas.suma/1
> f.(3)        ¡Ojo con el punto (.)! 
4
> g = &FuncionesBasicas.suma/2
> g.(1, 5)     ¡Ojo con el punto (.)! 
6

  • Se utiliza la siguiente sintaxis:

    var.(arg1, arg2, ..., argn)

¿Por qué es necesario ese punto?

  • Para evitar ambigüedades:
    • f.(x) invoca a la función almacenada en la variable f.
    • f(x) invoca a la función f/1 de ese mismo módulo, o de cualquier otro módulo que hayamos importado.
defmodule Fun do
  def f(x), do: x + 1

  def aplicar(f) do
    x = f.(0)    Llama a la función recibida como parámetro 
    y = f(3)     Llama a la función f/1 definida arriba 
    x + y
  end
end

Funciones de orden superior

Decimos que una función es de orden superior si recibe una o varias funciones como parámetro, o bien devuelve una función como resultado.

Ejemplo: Invocar una lista de funciones

  • La siguiente función apply_all/2, dada una lista de funciones fs y un parámetro x, devuelve una lista con los resultados de llamar a cada función de fs pasando x como parámetro:
def apply_all([], _), do: []
def apply_all([f | fs], x) do
  [f.(x) | apply_all(fs, x)]
end
> apply_all([&suma/1, &cuadrado/1, &IO.puts/1], 3)
3                                         Efecto lateral de IO.puts 
[4, 9, :ok]                               IO.puts devuelve :ok

Ejemplo: máximo de una lista

  • La función max, dada una lista no vacía, devuelve el máximo de los elementos de la lista.
def max([x]), do: x
def max([x | xs]) do
  max_tail = max(xs)
  if x >= max_tail, do: x, else: max_tail
end
> max([1, 5, 3, 2])
5
> max([{1, 5}, {2, 1}, {2, 4}, {1, 10}])
{2, 4}

  • La función anterior utiliza el operador >= para saber si un elemento es mayor que otro.

  • El operador >=, cuando se aplica a tuplas utiliza el orden lexicográfico.

    • Compara las primeras componentes.
    • En caso de igualdad, compara las segundas componentes.
    > {2, 5} >= {1, 4}
true
> {2, 5} >= {3, 4}
false
> {2, 6} >= {2, 3}
true

¿Y si quisieramos usar otra relación de orden?

  • Podemos hacer que max reciba un segundo parámetro: la función que se utilizará para comparar dos elementos.

    def max([x], _), do: x
def max([x | xs], mayor_que) do
  max_tail = max(xs)
  if mayor_que.(x, max_tail), do: x, else: max_tail
end

  • De hecho, podemos hacer que el valor por defecto del parámetro mayor_que sea la comparación mediante >=, que se denota mediante &>=/2.

    def max(xs, mayor_que \\ &>=/2)

  • Ejemplos:

    > max([{1, 6}, {5, 1}, {6, 4}], &comp_segunda_componente/2)
{1, 6}
> max([{1, 6}, {5, 1}, {6, 4}], &comp_suma/2)
{6, 4}

    donde comp_segunda_componente y comp_suma se definen del siguiente modo:

    def comp_segunda_componente({_, x}, {_, y}), do: x >= y

def comp_suma({x1, y1}, {x2, y2}), do: x1 + y1 >= x2 + y2

Funciones anónimas

Funciones anónimas

  • Hasta ahora, cada vez que asignábamos una función a una variable (o la pasábamos como argumento), suponíamos que la función estaba definida previamente mediante def.

    > f = &cuadrado/1
> g = &IO.puts/1
  • También es posible definir funciones en el mismo punto en el que se usan, sin necesidad de hacerlo mediante def.
    • Esto es útil para funciones de «un solo uso».
    • En este caso, las funciones definidas no tienen nombre, y reciben el nombre de anónimas.

  • Elixir proporciona dos mecanismos para definir funciones anónimas:

    • Notación fn ... -> ... end
    • Notación &(....)

Notación fn

  • Ejemplos de uso:

    > f = fn x -> x * x end   Función que calcula el cuadrado de un número 
> f.(4)
16
> g = fn x, y -> x + y end   Función que suma dos números 
> g.(1, 3)
4

  • En general, la sintaxis fn para definir funciones anónimas de aridad n es la siguiente:

    fn x1, x2, ..., xn -> expr end

  • Se permiten funciones anónimas de aridad cero:

    > f = fn -> IO.puts("Hola!") end
> f.()
Hola!
:ok

  • El cuerpo de la función anónima puede ocupar varias líneas:

    > f = fn x, y ->
        z = x + y
        z * z
      end
> f.(3, 5)
64

  • Se permite ajuste de patrones en los parámetros de la función anónima:

    > f = fn {x, y} -> {y, x} end
> f.({4, 5})
{5, 4}
> f.(4)
** (FunctionClauseError) no function clause matching in ...

  • También se permiten varias cláusulas:

    > f = fn 0 -> 1
         1 -> 0
         x -> :error
      end

> f.(1)
0
> f.(4)
:error

Ejemplo: max

  • En lugar de:

    > max([{1, 6}, {5, 1}, {6, 4}], &comp_segunda_componente/2)
{1, 6}
> max([{1, 6}, {5, 1}, {6, 4}], &comp_suma/2)
{6, 4}

    podría haberse escrito lo siguiente, en lugar de definir las funciones anteriores en un módulo.

    > max([{1, 6}, {5, 1}, {6, 4}], fn {_ x}, {_, y} -> x >= y end)
{1, 6}
> max([{1, 6}, {5, 1}, {6, 4}], 
      fn {x1, y1}, {x2, y2} -> x1 + y1 >= x2 + y2 end)
{6, 4}

Notación &(...)

  • Tiene la siguiente sintaxis:

    &(expr)

    donde expr es el cuerpo de la función que se define.

  • El cuerpo contiene marcadores de la forma &1, &2, etc., que hacen referencia a los parámetros.

Ejemplos

Notación fn

fn x, y -> x + y end
fn x -> x end
fn xs -> length(xs) + 1 end

Notación &(...)

&(&1 + &2)
&(&1)
&(length(&1) + 1)
  • En algunos casos es posible omitir los paréntesis:
&{&1, &2}         &[0 | &1]        & &1

Ejemplo

> apply_all([&(&1 + 1), &(&1 * &1), &to_string/1], 3)
[4, 9, "3"]

Limitaciones de la notación &(...)

  • No se permiten funciones de aridad cero:

    • Incorrecto: &(2)
    • Correcto: fn -> 2 end

  • No se permite ajuste de patrones sobre los parámetros:

    • Por ejemplo, si queremos algo equivalente a

      fn {x, y} -> {y, x} end

      tendríamos que escribir:

      &{elem(&1, 1), elem(&1, 0)}

Limitaciones de la notación &(...)

  • El cuerpo de la función anónima ha de ser una expresión simple. No se permiten bloques de expresiones.

    • Incorrecto:

      &(y = cuadrado(&1)
  y + 1)

    • Correcto:

      &(cuadrado(&1) + 1)

Operador pipe (|>)

Operador pipe (|>)

  • El operador de tubería (pipe operator) se utiliza con la siguiente sintaxis:

    expr |> fun(arg1, arg2, ..., argn)

    donde n ≥ 0.

  • La aplicación de este operador es equivalente la siguiente expresión:

     fun(expr, arg1, arg2, ..., argn)

Aplicaciones

  • Este operador es muy utilizado para expresar la aplicación de un pipeline de operaciones donde el resultado de una operación sirve como entrada para la siguiente:
def magnificar(str) do                               
  str                                               "   Es un buen invento  " 
  |> String.trim()                                  "Es un buen invento" 
  |> String.downcase()                              "es un buen invento" 
  |> String.replace(~r{buen(o)?}, "maravilloso")    "es un maravilloso invento" 
  |> String.upcase()                                "ES UN MARAVILLOSO INVENTO"     
  |> Kernel.<>("!!")                                "ES UN MARAVILLOSO INVENTO!!" 
end
> magnificar("  Es un buen invento  ")
"ES UN MARAVILLOSO INVENTO!!"
  • También puede utilizarse el operador pipe con funciones anónimas expresadas mediante fn.
  • En este caso, es necesario poner el punto (.) antes de los paréntesis de la llamada.
def magnificar(str) do                               
  str
  |> String.trim()
  |> String.downcase()
  |> String.replace(~r{buen(o)?}, "maravilloso")
  |> String.upcase()
  |> (fn str -> "¡¡#{str}!!" end).()
end
> magnificar("  Es un buen invento  ")
"¡¡ES UN MARAVILLOSO INVENTO!!"
