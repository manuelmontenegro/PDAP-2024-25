Operador de ajuste (=) y tipos de patrones

Operador de ajuste (=)

  • Hasta ahora hemos utilizado el operador para asignar un valor a una variable.

    Por ejemplo, x = 5

  • Sin embargo, el lado izquierdo del operador = puede ser más complejo que una variable.

  • La forma general del operador = es la siguiente:

    patron = expresion

Tipos de patrones

  • Constantes
  • Variables
  • Patrones tupla
  • Patrones lista
  • Patrones string
  • Variables anónimas (_)
  • Variable con operador pin
  • …

Patrón constante

  • Es posible colocar un número o un átomo en el lado izquierdo de un ajuste de patrones.
c = exp

  • Si la expresión exp se evalúa a un valor distinto de c, el ajuste falla.

  • En caso contrario, el ajuste tiene éxito.

Ejemplos:

> y = 3
3
> x = 2
2
> 3 = y            OK 
3
> 2 = y            Fallo 
** (MatchError) no match of right hand side value: 3 
> 3 = x + 1        OK 
3
> 1 = x            Fallo 
** (MatchError) no match of right hand side value: 2
> 5 = :hola        Fallo 
** (MatchError) no match of right hand side value: :hola
> :hola = :hola    OK 
:hola

Patrón variable

  • Si en el lado izquierdo de un ajuste de patrones es una variable, el ajuste siempre tiene éxito.

  • Además, la variable del lado izquierdo queda asignada al valor del lado derecho.

> x = 4
4
> x
4
> y = 7
7
> y = 3
3
> y
3

Patrón tupla

  • Los patrones tupla tienen la forma {pat1, …, patn}.

    {pat1, pat2, ..., patn} = exp

  • El ajuste de patrones tiene éxito si y solo si la expresión exp se evalúa a una tupla de n componentes, y además:

    • La primera componente ajusta con pat1,
    • La segunda componente ajusta con pat2,
    • …
    • La n-ésima componente ajusta con patn.

Ejemplo:

> z = {3, 4}
> {x, y} = z       OK 
> x
3
> y
4

Más ejemplos:

> {x, y} = :hola          Fallo. El lado derecho no es una tupla 
> {x, y, z} = {1, 4}      Fallo. El lado derecho no tiene 3 componentes 
> {x, y} = {1, 4}         Correcto. La x queda ligada al valor 1; la y, al 4 
> {1, 3} = {0, 3}         Fallo. El 0 no encaja con el 1 
> {1, 3} = {x, 3}         Correcto, porque x tiene el valor 1 
> {x, 4} = {2, y}         Correcto. La x ahora queda ligada al valor 2 
> x
2
> y
4

Patrón lista

  • Los patrones de este tipo pueden ser la lista vacía [], o bien tener la forma [pat1 | pat2].

    1. [] = exp

    El ajuste tiene éxito si y solo si exp se evalúa a una lista vacía.

    1. [pat1 | pat2] = exp

    El ajuste tiene éxito si y solo si exp se evalúa a una lista no vacía, y además:

    • la cabeza de la lista ajusta con pat1, y

    • la cola de la lista ajusta con pat2.

Ejemplos:

> z1 = []
> z2 = [5, 3, 1]
> [x | y] = z1       Fallo 
> [x | y] = z2       OK. x = 5, y = [3, 1] 
> [x | y] = [6]      OK. x = 6, y = [] 
> x
6
> y
[]

Ejemplos:

> [{x, y} | z] = [2, 5, 7]      Fallo. El 2 no encaja con {x, y} 
> [{x, y} | z] = [{1, 1}, 4]    OK. x = 1, y = 1, z = [4] 
> [[1 | v] | w] = [4, 5, 10]    Fallo. El 4 no encaja con [1 | v] 
> [[1 | v] | w] = [[4, 5], 10]  Fallo, el 4 no encaja con 1 
> [[1 | v] | w] = [[1, 5], 10]  OK. v = [5], w = [10] 
> [x | [y | z]] = [1, 4, 5]     OK. x = 1, y = 4, z = [5] 
> [x | [y | z]] = [1]           Fallo. [] no encaja con [y | z] 
> [x | [y | z]] = [1, 3]        OK. x = 1, y = 3, z = []

  • Los patrones lista admiten el mismo azucar sintáctico de las listas. En particular:

    • [pat1, pat2, …, patn | patn+1] equivale a
      [pat1 | [pat2 | … [patn | patn+1] … ]]

    • [pat1, pat2, …, patn] equivale a
      [pat1 | [pat2 | … [patn | []] … ]]

Ejemplos:

> [x | y] = [1, 2]           OK. x = 1, y = [2] 
> [x, y] = [1, 2]            OK. x = 1, y = 2 
> [1, 2 | z] = [1, 2, 4, 6]  OK. z = [4, 6] 
> [x, y] = [1, 2, 4, 6]      Fallo. [4, 6] no encaja con [] 
> [v] = [3]                  OK. v = 3 
> [v] = [3, 4]               Fallo. [4] no encaja con []

Patrón string

  • Hay dos tipos:

    1.    "cad" = exp

      Tiene éxito si y solo si exp se evalúa a la cadena "cad".

    2.    "cad" <> pat = exp

      Tiene éxito si y solo si exp se evalúa a una cadena que comienza por "cad". El resto de la cadena se ajusta al patrón pat.

Ejemplos:

> "hola" = "hola"          OK 
> "h" <> x = "hola"        OK. x = "ola" 
> "hol" <> x = "hola"      OK. x = "a" 
> "hola" <> x = "hola"     OK. x = "" 
> "hola" <> x = "adios"    Fallo 
> "hola" <> x = "hol"      Fallo

Inciso: variables repetidas

  • Cuando utilizamos el operador =, una misma variable puede aparecer varias veces en el patrón.

  • En ese caso, todas las apariciones de esa variable han de quedar ligadas al mismo valor. En caso contrario, el ajuste de patrones fallará.

{x, x} = {1, 3}                          Fallo. ¿x = 1 o x = 3? 
{x, y, x} = {2, 5, 2}                    OK. x = 2, y = 3 
[[x | y], [x | z] | v] = [[2, 5], [2]]   OK. x = 2, y = [5], z = [], v = []

Variables anónimas

  • Se representan mediante el guión bajo (_).

  • Ajustan con cualquier valor, al igual que un patrón variable, pero no se ligan a ningún valor.

{x, _, 5} = {3, 1, 5}      OK. x = 3 
[x, y | _] = [1, 2, 3]     OK. x = 1, y = 2 
{_, _, _} = {1, 8, 3}      OK 
{x, x, x} = {1, 8, 3}      Fallo

Patrones con operador pin (^)

  • Tienen la forma ^x, donde x es una variable.
  • Encajan con aquellos valores que coincidan con el valor de la variable x.
  • No hacen que la variable x cambie el valor al que está ligada.
> {x, y} = {4, 5}          OK. x = 4, y = 5 
> ^x = 3                   Fallo. El 3 no encaja con el valor de x (que es 4) 
> x = 3                    OK. x = 3 
> {^x, ^y} = {3, 4}        Fallo. El 4 no encaja con el valor de y (que es 5) 
> {^x, ^z} = {3, 4}        Error de compilación. La z no tiene ningún valor 
> {^x, z} = {3, 4}         OK. z toma el valor 4

Mecanismos de ajuste de patrones

Mecanismos de ajuste

  • Operador de ajuste (=)
  • Parámetros de las funciones
  • Expresiones case

Operador de ajuste (=)

  • Como ya hemos visto, el operador = solamente tiene éxito si la expresión del lado derecho se evalúa a un valor que encaje con el patrón del lado izquierdo.

  • El operador = se utiliza habitualmente cuando se tiene la certeza de que el ajuste de patrones va a tener éxito y, en caso de no ser así, queremos que el programa falle.

    def suma(t) do
  {x, y} = t
  x + y
end

    Suponemos que suma siempre va a ser invocada con una tupla de dos componentes como argumento.

Ajuste mediante parámetros

  • Es posible introducir patrones en los parámetros de las funciones.

  • Por ejemplo, en lugar de

    def suma(t) do
  {x, y} = t
  x + y
end

    puede escribirse

    def suma({x, y}) do
  x + y
end
  • Cuando llamamos a una función que consta de varias cláusulas, se ejecutará la primera cláusula en la que los patrones encajen con los argumentos de la llamada y se cumpla la guarda de la cláusula.
def cuadrante({x, y}) when x > 0 and y > 0, do: :primero
def cuadrante({x, y}) when x < 0 and y > 0, do: :segundo
def cuadrante({x, y}) when x < 0 and y < 0, do: :tercero
def cuadrante({x, y}) when x > 0 and y < 0, do: :cuarto
def cuadrante({0, 0}), do: :origen
def cuadrante({x, y}), do: :eje
def cuadrante(_), do: {:error, :invalid_argument}
> cuadrante({1, 3})
:primero
> cuadrante({0, -5})
:eje
> cuadrante({1, 4, 6})
{:error, :invalid_argument}

  • En la siguiente cláusula del ejemplo anterior

    def cuadrante({x, y}), do: :eje

    no estamos utilizando los valores de x e y en el cuerpo de la función.

  • El compilador muestra un aviso indicando que es preferible utilizar variables anónimas en esos casos:

    def cuadrante({_, _}), do: :eje

  • La última cláusula del ejemplo anterior es una cláusula catch-all que devuelve una tupla que indica que el argumento es inválido.

    def cuadrante(_), do: {:error, :invalid_argument}

  • En general, si solamente tenemos pensado llamar a cuadrante con tuplas de dos componentes, es preferible evitar este tipo de cláusulas.

    • De este modo, si por error llamamos a cuadrante con otra cosa distinta, el programa fallará, haciendo el error explícito.
def cuadrante({x, y}) when x > 0 and y > 0, do: :primero
def cuadrante({x, y}) when x < 0 and y > 0, do: :segundo
def cuadrante({x, y}) when x < 0 and y < 0, do: :tercero
def cuadrante({x, y}) when x > 0 and y < 0, do: :cuarto
def cuadrante({0, 0}), do: :origen
def cuadrante({x, y}), do: :eje
> cuadrante({1, 4, 6})
** (FunctionClauseError) no function clause matching in Cuadrante.cuadrante/1

Expresiones case

  • La expresión case permite comparar un valor con una serie de patrones hasta encontrar uno que ajuste con el valor proporcionado:

    def suma(t) do
  case t do
    {x, y} -> x + y
    {x, y, z} -> x + y + z  
  end
end
    > suma({1, 4})
5
> suma({3, 5, 1})
9

  • En general, una expresión case tiene la siguiente forma:

    case exp do
  pat1 when guarda1 -> exp1
  pat2 when guarda2 -> exp2
  ...
  patn when guarda2 -> expn
end

  • Se evalúa la expresión exp y se ejecuta la primera cláusula i tal que exp encaje con pati y se cumpla guardai.

  • Las guardas son opcionales.

  • Si la expresión exp no ajusta con ninguno de los patrones de las cláusulas, se produce un error.

  • Es posible introducir una variable anónima como patrón catch-all:

    def segundo_elemento(l) do
  case l do
    [_, y | _] -> y
    _ -> nil
  end
end
    > segundo_elemento([:a, :b, :c])
:b
> segundo_elemento([:a])
nil

Diferencia entre case y funciones con múltiples cláusulas

  • Si el cuerpo de una función consiste simplemente en un case, puede escribirse como una función con múltiples cláusulas:

  • Por ejemplo, la función segundo_elemento mostrada anteriormente se puede implementar del siguiente modo:

    def segundo_elemento([_, y | _]), do: y
def segundo_elemento(_), do: nil
  • Las expresiones case sí son útiles cuando el cuerpo de la función realiza otras operaciones, además del case.
def comprueba_punto(x) do
  c = cuadrante(x)
  case c do
    :primero -> true
    _ -> false
  end
end
def comprueba_punto(x) do
  case cuadrante(x) do
    :primero -> true
    _ -> false
  end
end
